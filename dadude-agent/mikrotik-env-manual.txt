# ==========================================
# COMANDI MANUALI PER ENVIRONMENT VARIABLES
# ==========================================
#
# RouterOS potrebbe avere una sintassi diversa per /container/envs/add
# Prova questi comandi uno alla volta:
#
# 1. Verifica la sintassi esistente:
#    /container/envs/print detail
#
# 2. Prova senza "name=":
#    /container/envs/add key=DADUDE_SERVER_URL value="https://dadude.domarc.it:8000"
#
# 3. Se funziona, continua con gli altri:
#    /container/envs/add key=DADUDE_AGENT_TOKEN value="mio-token-rb5009"
#    /container/envs/add key=DADUDE_AGENT_ID value="agent-rb5009-test"
#    /container/envs/add key=DADUDE_AGENT_NAME value="RB5009 Test"
#    /container/envs/add key=DADUDE_DNS_SERVERS value="192.168.4.1,8.8.8.8"
#    /container/envs/add key=PYTHONUNBUFFERED value="1"
#
# 4. Poi trova l'ID dell'envlist creata:
#    /container/envs/print
#
# 5. Aggiorna il container con l'envlist:
#    /container/set 0 envlist=[ID trovato sopra]
#
# 6. Avvia il container:
#    /container/start 0
#
# ALTERNATIVA: Se /container/envs/add non funziona, prova a passare
# le env direttamente nel comando del container modificando il cmd:
#    /container/set 0 cmd="DADUDE_SERVER_URL=https://dadude.domarc.it:8000 DADUDE_AGENT_TOKEN=mio-token-rb5009 python -m app.agent"
#
